**рҹ“Ң About the Tool**

This script was developed to automate and optimize the hunting process for credential and token leaks in public Postman workspaces. Using the base tool postleaks, this script not only performs mass searches based on a wordlist but also intelligently manages the results for a Continuous Recon workflow.

The core feature of this automation is the Hash-based tracking system. It remembers which leaks you have already analyzed/reported and only brings up new (unseen) results, saving triage time and preventing false positives week after week.

**рҹҡҖ How to Use**

```bash
./postman_recon.sh
```

The script will launch an interactive menu with 3 options:

**1. Hunting**

In this step, the script will prompt you for your keywords file (e.g., targets.txt) and the client name (which will be used to create the target directory).

 * It reads your wordlist line by line.
 * Executes postleaks for each keyword in an isolated temporary folder.
 * Renames the random JSON outputs generated by Postman into the respective keyword name to make identification easier (e.g., client_com_br.json), and saves everything inside the client's folder.

**2. Show New Results**
   
This is the triage and visualization stage. The script will ask for the client folder you want to analyze.

* **Clean Extraction**: It scans all JSON files and prints only the relevant data (Name, Method, URL, and Headers), fully colorized and formatted.

* **Smart Filtering**: It generates a "Footprint" (Method + URL + Headers) for each leak and hashes it using MD5. If this hash already exists in the hidden .known_hashes.txt file, the script ignores it. You will only see brand-new findings on your screen.

* **Confirmation**: At the end of the output, the script will ask: "Do you want to mark these results as reported?". If you answer y (yes), the new hashes are saved to the database. Next week, they won't show up again unless the developer changes the URL, Method, or Token inside Postman.

**3. Exit**

Closes the tool cleanly.


**рҹ’Ў Useful Tips**

* **View hidden files**: To view the client's hash database, run: 

```bash
ls -la client_name/
```

* **Reset history**: If you want the tool to "forget" what was reported and show ALL results from scratch, simply delete the hidden hash file:

  
```bash
rm client_name/.known_hashes.txt
```

**Execution permissions**: If you get a permission denied error when trying to run the script for the first time, don't forget to grant execution rights: 

```bash
chmod +x postman_recon.sh
```
