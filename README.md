**рЯУМ About the Tool**

This script was developed to automate and optimize the hunting process for credential and token leaks in public Postman workspaces. Using the base tool postleaks, this script not only performs mass searches based on a wordlist but also intelligently manages the results for a Continuous Recon workflow.

The core feature of this automation is the Hash-based tracking system. It remembers which leaks you have already analyzed/reported and only brings up new (unseen) results, saving triage time and preventing false positives week after week.

1. Hunting
In this step, the script will prompt you for your keywords file (e.g., targets.txt) and the client name (which will be used to create the target directory).

It reads your wordlist line by line.

Executes postleaks for each keyword in an isolated temporary folder.

Renames the random JSON outputs generated by Postman into the respective keyword name to make identification easier (e.g., alelo_com_br.json), and saves everything inside the client's folder.
